Hydra

What is Hydra?

Hydra is a powerful tool that comes preinstalled on Kali Linux and is used to perform Brute-Force attacks.

Protocols Supported by Hydra:

Asterisk, AFP, Cisco AAA, Cisco auth, Cisco enable, CVS, Firebird, FTP, HTTP-FORM-GET, HTTP-FORM-POST, HTTP-GET, HTTP-HEAD, HTTP-POST, HTTP-PROXY, HTTPS-FORM-GET, HTTPS-FORM-POST, HTTPS-GET, HTTPS-HEAD, HTTPS-POST, HTTP-Proxy, ICQ, IMAP, IRC, LDAP, MEMCACHED, MONGODB, MS-SQL, MYSQL, NCP, NNTP, Oracle Listener, Oracle SID, Oracle, PC-Anywhere, PCNFS, POP3, POSTGRES, Radmin, RDP, Rexec, Rlogin, Rsh, RTSP, SAP/R3, SIP, SMB, SMTP, SMTP Enum, SNMP v1+v2+v3, SOCKS5, SSH (v1 and v2), SSHKEY, Subversion, Teamspeak (TS2), Telnet, VMware-Auth, VNC and XMPP.

Hydra Installation

We go to the Github repository at https://github.com/vanhauser-thc/thc-hydra and copy the link from there and clone it to our system. We perform this operation by specifying the location where you want to clone.

After the cloning process is finished, you can install it with the "apt-get install hydra" command.

When we come to the terminal and type hydra, we will see the output of informative texts for use.

For more detailed information, we can take a look at the information below by typing “hydra -help” into the terminal.

Hydra Attack Phase and Use

In the output of Hydra, which is shown to us with the help command, 4 different attack examples are shown, but these attacks can be multiplied.

Different protocols are attacked differently using the Hydra tool.

• If we want to do a brute force attack on the FTP service and we know the username, we can use the following command.

hydra -l username -P /PATH/passwordlist.txt ftp://IP

• If we want to make a brute force attack on an SSH service, the command we will use is as follows;

hydra -l username -P /PATH/passwordlist.txt IP -t 4 ssh

In order for these attacks to be organized, it is necessary to know what type of request (GET or POST) is being made. In order to know this request, the source code of the page is accessed through the browser and it is understood what kind of request it is.

If we come across a login form using a POST prompt, the command we will use will be as follows:

hydra -l -P http-post-form "/:username=^USER^&password=^PASS^:F=incorrect" -V
