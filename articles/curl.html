cURL

cURL, short for 'Client URL', is open source software that runs on the terminal to send and receive files, check the connectivity of URLs and transfer data.

Supported Protocols:

• HTTP and HTTPS
• FTP and FTPS
• IMAP and IMAPS
• POP3 and POP3S
• SMB and SMBS
• SFTP
• SCP
• TELNET
• GOPHER
• LDAP and LDAPS
• SMTP and SMTPS

The commands in the list are the most important protocols, although there are other supported commands.

cURL Version Control:

curl -–version

Using cURL

cURL's simple syntax is as follows:

curl [OPTIONS] [URL]

Simple use of curl displays the contents of a page. Unlike browsers, cURL cannot handle HTML and cannot run JavaScript.

The following example will show the contents of the home page of google.com:

curl http://google.com

We can view raw HTTP requests using the “-v” switch to increase the level of detail.

cURL – Basic AUTH Login

curl http://guest:guest@192.168.1.1:1234/ -vvv

We can do a simple login with the example command above.

curl -u guest:guest http://192.168.1.1:1234/ -vvv

Alternatively, we can specify our credentials with the -u command.

curl -u guest:guest -L http://192.168.1.1:1234/

Unlike browsers, cURL does not redirect us to the specified location by default. The "-L" command tells cURL to follow redirects.
